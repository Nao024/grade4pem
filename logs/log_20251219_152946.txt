=== 解析結果 ===
原因
今の書き方だと、2つ目の数がいつまでも同じままです。1回目の計算で1つ目の数は小さくなりますが、その次から同じ組み合わせが続いてしまい、終わらなくなります（その結果、エラーになります）。

最も重要なエラー
次に呼ぶときに渡す2つの数の順番が逆になっていること。

修正方法
次に計算するときは「2つ目の数」と「1つ目の数を2つ目の数で割ったときのあまり」を、この順番で渡します。つまり
- 今: gcd(a % b, b)
- 正しくは: gcd(b, a % b)

修正後のコード例（必要最小限の変更）
public static int gcd(int a, int b) {
    if (b == 0) {
        return a;
    } else {
        return gcd(b, a % b);
    }
}

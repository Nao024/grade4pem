=== 解析結果 ===
原因と直し方
- 問題の核心: 自分自身を呼ぶときに、渡している数の順番が間違っているため、2つ目の数がずっと変わらず、いつまでも呼び出しが止まらなくなっています。その結果、メモリが溢れてエラー（StackOverflowError）になります。
- 直し方: 次に渡す数の順番を「b, aのbで割ったあまり」に変えます。これで2つ目の数がだんだん小さくなり、最後は0になって正しく終わります。

修正後のコード（GCD.java の gcd メソッドのみ）
public static int gcd(int a, int b) {
    if (b == 0) {
        return a;
    } else {
        return gcd(b, a % b);
    }
}

=== 解析結果 ===
原因（最も重要なエラー）
・計算を進めるときの「入れ替え」が間違っています。今のままだと、同じ値を片方に持ったまま自分を呼び続けてしまい、いつまでも終わりません。

直し方（専門用語を避けて一言で）
・呼び出しの順番を入れ替えてください。

具体的な修正
・GCD.java の最後の行を、次のように1行だけ直します。
  修正前: return gcd(a % b, b);
  修正後: return gcd(b, a % b);

・（おまけの安全策）0のときに返す値は正にしておくと安心です。
  if (b == 0) return Math.abs(a);

修正後のメソッド例
public static int gcd(int a, int b) {
    if (b == 0) {
        return Math.abs(a);
    } else {
        return gcd(b, a % b);
    }
}

なぜこれで直るか（やさしく）
・2つの数を「入れ替えながら小さくしていく」ことで、必ずどちらかが0になります。0になったらもう一方が答えなので、きちんと止まります。
